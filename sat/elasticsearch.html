ELK Stack

ELK is the acronym for Elasticsearch, Logstash and Kibana.

Elasticsearch

Elasticsearch/Elastic stack is a distributed, open source, search and analytics engine for all types of data, including textual, numerical, structured and unstructured. It's a RESTful application. Elasticsearch is built on Apache Lucne and it's famous for speed and scalability. It is basically a document store like MongoDB. Data is stored as "Schema-less" JSON objects (like No SQL documents). It provides near real-time search (meaning there is only a small latency from a document is indexed untill it is searchable).


Communication to server is done through HTTP REST API. Title: Unrefactored CodeLanguage: Shell curl =X {REST Verb (GET/POST)} {Node}:{Port}/{Index}/{Type}/{ID}

Speed:

Elasticsearch uses inverted indices for full text querying, BKD trees for storing numeric and geo data, and a column store for analytics.

Scalability:

Run it on your laptop or huundreds of server with petabytes of data. Single node runs the same way as your in a 300-node cluster.

Relevance

Can handle complex typos and ranks the search results based on variety of factors from frequencty or recency to popularity and beyond.

Resiliency

Detects failures to keep your cluster safe and available. Operates on a distributed environment.

Kibana:

Its an analytics and visualization platform. Kibana can be treated as elastic search dashboard, providing line chart, pie chart and many others. Multiple dashboards can be created for different dashboards. It can be also used to search the elasticsearch logs.

Logstash:

Its a data processing pipeline use to send the information to elasticsearch. Logstash has evolved to support multiple applications other than elasticsearh in recent times. Data logstash receives are treated as events and these events can be sent multiple destinations like elasticsearch, email, kafka. Logstash has 3 parts. 1. Inputs, 2. Filters and 3. output olugins.


Logstash => Ingestion          Elasticsearch => Searching          Kibana => Visualization

Elasticsearch is the heart of elastic stack but elastic stack so most of the time Elastic stack is refereed as Elasticsearch. Elastic stack includes other tools which is optionally used with elasticsearch. Below are those components of elastic stack:

  • X-Pack:

    Provides authentication and authorization for Kibana and Elastic search. Supports LDAP and roll based configuration. It also providesalerting. Reports can be generated automatically or manually. X-Pack also provides Machine learning feature.

  • Graph:

    Identify relevance between the data. Integrated into applications using API and also can be visualized using Kibana.

  • SQL:

    Elasticsearch queries are written using Query DSL(proprietery Query language). But DSL is very verbose. So Elasticsearch SQL provides an API to write SQL query and it internally converts into Query DSL.

  • Beats:

    Beats are data shippers. They basically contain bunch of light weight tools which can be installed in your application to send data to Logstash or Elasticsearch.
    • FileBeat: Used to collect log files from Apache webserver or MySQL or Kibana or Postgres and send it to Elasticsearch.
    • MtricBeat: Collects system and service information E.g. CPU and Memory usages
    • PacketBeat: Colelcts network data E.g. Http Requests
    • WinLogBeat: Collects Windows Log Events
    • AuditBeat: Collects audit data from Linux
    • HeartBeat: Monitors Service Uptime


Applications of elastic search:

  1. Logging:

    ELK is the acronym for Elasticsearch, Logstash and Kibana.

    Provides fast and scalable logging with out-of-the-box support for the common data sources(Kubernetes, Apache, EC2, MySQL, Windows, etc.). Index your data into elasticsearch and visualize it all in Kibana in minutes.

    Some of the out-of-the-box features provided for logging are listed below:
    • Tail a file directly in the UI

    • Add machine learning to automate anamoly detection

  2. Metrics:

    Monitor and visualize the system metrics in one place. Metric beat module will help to get started in minutes to track Kubernetes orchestratioin or prometheus-style metrics.
    • Deep dive into metrics to compare current and historical CPU/memory/network traffic time series data.
    • Consolidate data from hundreds of Prometheus server and acheve global view and analyze them in Kibana. Metricbeat can be connected to the Prometheus server.
    • Search across high cardinality data with the visualization

  3. Application Performance monitoring(APM):

    Monitor application metrics to identify where your applicaiton is spending time.
    • By providing one stop solution for reports, transactions, logs and metrics, it makes the root cause analysis easier.
    • Detect anamolies or anamolous response times by enabling machine learning-based anamoly detection feature
    • Get alerted based on severity or threshold. Get notifications via email, slack, etc.
    • Supports most of the popular programming language applications.

  4. Service Uptime:

    Monitor uptime and react to availability issues across apps and services.
    • Active monitoring with custom alerting options
    • Supports multistep check validation

  5. Site Search:

    provides the tools to build powerful site search. Provides web crawling and searching feature.
    • Capture content changes automatically
    • Flexibility to manually reindex specific pages
    • Analysse search behavior in real time from built-in analytics dashboards.
    • Deliver faster and better results using elastic search

  6. App Search:

    provides focused power of elastic search to apps with redefined set of APIs and intutive dashboards.

  7. Map:

    Explore geospatial data with Elastic maps.
