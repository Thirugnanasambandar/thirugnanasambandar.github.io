Elasticsearch

Elasticsearch is a distributed, open source, search and analytics engine for all types of data, including textual, numerical, structured and unstructured. It's a RESTful application. Elasticsearch is built on Apache Lucne and it's famous for speed and scalability.

Speed:

Elasticsearch uses inverted indices for full text querying, BKD trees for storing numertic and geo data, and a column store for analytics.

Scalability:

Run it on your laptop or huundreds of server with petabytes of data. Single node runs the same way as your in a 300-node cluster.

Relevance

Can handle complex typos and ranks the search results based on variety of factors from frequencty or recency to popularity and beyond.

Resiliency

Detects failures to keep your cluster safe and available. Operates on a distributed environment.

UseCases:

  1. Logging:

    Provides fast and scalable logging with out-of-the-box support for the common data sources(Kubernetes, Apache, EC2, MySQL, Windows, etc.). Index your data into elasticsearch and visualize it all in Kibana in minutes. Some of the out-of-the-box features provided for logging are listed below:
    • Tail a file directly in the UI
    • Analyze trends with categorized logs
    • Flexible data stream processing using Beats and ingest node
    • Add machine learning to automate anamoly detection

  2. Metrics:

    Monitor and viisualize the system metrics in one place. Metricbeat module will help to get started in minutes to track Kubernetes orchestratioin or prometheus-style metrics.
    • Deep dive into metrics to compare current and historical CPU/memory/network traffic time series data.
    • Consolidate data from hundreds of Prometheus server and acheve global view and analyze them in Kibana. Metricbeat can be connected to the Prometheus server.
    • Search across high cardinality data with the visualization
